# Wolffia Case Study

Use this file only when the task is specifically about `Wolffia australiana` or when the user wants a concrete plant example of how the generic `PIP-seq` workflow can be adapted.

## Why it matters

`Wolffia australiana` is unusually small and morphologically reduced, so some preparation logic that looks strange in a generic protocol can make sense here.

## What this case study supports

- tiny-plant handling language
- pooled frond or whole-plant collection logic
- waterproof-cuticle rationale for gentle dicing before digestion
- Wolffia-specific interpretation of simplified or merged cell programs

## What it does not support

- universal dissociation parameters for all plants
- universal expectations about cell-type complexity
- universal conclusions about how `PIP-seq` should be run in any species

## Best use

Use Wolffia as:

- a worked organism-specific benchmark
- a test case for compressed or merged plant programs
- a source of practical wording for tiny aquatic plant preparation

Do not present Wolffia-specific workflow choices as default plant workflow choices.
